Library

라이브러리는 소프트웨어의 구성요소 중 하나로, API를 기반으로 대상 환경(플랫폼)에서 바로 실행될 수 있도록 모듈화된 프로그램 모음이다.
라이브러리는 혼자서 동작하는 완전한 프로그램이 아닌, 특정한 부분 기능만으로 수행하도록 제작된, 컴파일되어 기계어의 형대로 존재하는 프로그램이다.

  • 표준 라이브러리
    • 특정 언어의 개발 환경에 기본적으로 포함된 것들은 표준 라이브러리라고 불린다. 기본적으로 기능수행과 더불어 디버깅, 성능측정 등을 위한 별도의 API가 존재한다.
  • 런타임 라이브러리
    • 프로그램이 실제 환경에서 실행되기 위해 필요한 모듈들이다. 대부분 위의 표준 라이브러리에서 기능 수행에 필요한 것들만 제공되거나, 스크립트의 실행기 등을 말한다.

많은 회사나 개인, 단체들이 상용 혹은 오픈 소스로 라이브러리들을 개발하고 배포하고 있기에, 이를 활용하여 개발에 사용하면 필요한 시간과 노력을 절약할 수 있다.

Framework

프레임워크는 어떠한 목적을 달성하기 위해 복잡하게 얽혀 있는, 문제를 해결하기 위한 구조이며, 소프트웨어 개발에 있어서 하나의 뼈대 역할을 한다.

Web Framework

웹 프레임워크 또는 웹 애플리케이션 프레임워크는 웹 서비스 개발을 위한 프레임워크이다. 대표적인 프레임워크로는 Spring, Django, Node.js, Ruby on Rails 등이 있다.

Front-end Framework
React

React는 메타에서 개발한 오픈 소스 JavaScript 라이브러리이다.
프론트엔드 개발자 사이에서 많은 인기를 얻고 있고, 이는 Github Star 수와 npm 패키지 다운로드 수가 증명한다.

SPA을 전제로 하고 있으며, Dirty checkingRHK DOM을 활용하여 업데이트 해야하는 DOM 요소를 찾아서 해당 부분만 업데이트하기 때문에, 리렌더링이 잦은 동적인 모던 웹에서 엄청나게 빠른 퍼포먼스를 내는게 가능하다. 기본적으로 모듈형 개발이기 때문에 생산성 또한 상당히 높다. 특히 프레임워크가 아닌 프레임워크이기 때문에 다른 프레임워크에 간편하게 붙여서 사용하는 것도 가능하며, React Hooks라는 강력한 메소드를 지원하면서 사실상 웹 프론트엔드 개발의 표준으로 자리잡았다. TypeScriptsk Sass 같은 언어도 지원하고, Next.js 등의 등장으로 인해 사용 범위 또한 기하급수적으로 늘어났다.

또한 React Native라는 프레임워크를 사용하면 React의 문법으로 안드로이드, iOS 앱을 개발할 수 있다. React와 거의 유사한 문법을 가지고 있어 익숙해지기 매우 쉽다는 것이 특징. 이를 활용하면 크로스 플랫폼 개발을 편하게 진행할 수 있다. 이로 인해 크로스 플랫폼 개발에선 Google의 Flutter와 양분하고 있다고 볼 수 있다.

Vue.js

Vue는 JavaScript로 개발된 컴포넌트 구조 기반 프론트 엔드 프레임워크이다.

Vue는 뛰어난 내장 응용 프로그램을 가지고 있어 빠른 속도를 가지고 있다. 특히 스마트폰 분야에서는 React보다 빠르게 프로젝트를 빌드한다. 또한 다른 프레임워크와 달리 유연하고 가볍다는 것도 특징이다.

Angular

Angular는 구글에서 운용, 유지 중인 JavaScript 웹 프레임워크이다.

React, Vue보다 학습 시간이 길어 인지도가 둘에 비해 현저하게 낮은 편이다.

Back-end Framework
Spring

Spring는 Java/Kotlin 기반의 웹 프레임워크이다.

Spring은 종속 객체를 생성해주고, 조립할 수 있는 프레임 워크이다. 또한 자바SE로 된 자바 객체(POJO)를 자바EE에 의존적이지 않게
연결해준다. 주목할 점은 한국 전자정부표준프레임워크의 기반 기술이며 한국정보화진흥원에서는 공공기관의 웹 서비스 제공시 스프링을
권장하고 있다.

Nest.js

Nest.js는 Node.js 런타임 위에서 동작하는 TypeScript용 오픈 소스 백엔드 웹 프레임워크이다.

Node.js 생태계에서 많이 쓰이는 Express.js에 비하여 제어 역전성을 가지는 프레임워크적 성향이 더욱 강하며, 자바 생태계의 Spring와 자주
비교된다. 다만 Nest.js 자체는 독립적인 프레임워크가 아니며, 내부적으로 기존의 웹 프레임워크들의 기능을 일관된 API로 노출해 주는 거대한
어댑터(Adapter)에 가깝다.

Django

Django는 2005년부터 시작된 Python의 오픈 소스 웹 프레임워크이자 풀 스택 프레임워크이다.

Django는 Python 기반 웹 프레임워크 중에 가장 널리 퍼져있다. 일반적으로 파이썬을 웹 서비스에 쓴다고 한다면 Django 또는 Flask를 쓴다고 볼 수 있을 정도이다. Flask는 마이크로 프레임워크라 볼 수 있기에 풀 스택 프레임워크 중에서는 가장 많이 쓰는 파이썬 프레임워크라 볼 수 있다.

Ruby on Rails

Ruby on Rails는 Ruby를 사용하는 오픈소스 풀스택 웹 프레임워크로 덴마크의 David Heinemeier Hansson이 만들었다.

Ruby on Rails 이후 등장한 거의 모든 풀스택 웹 프레임워크에 영향을 주었다고 과연이 아니다. Django, Laravel 등이 이 Rails에 영향을 받아 설계 되었다. Rails의 특징으로는 언어의 특성에 의해 생산성이 높고, 확장성과 이식성 또한 높다. 또한 세부적인 설정이 적은 편이다.
