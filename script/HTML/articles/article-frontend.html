Front-end & Back-end

프로그래밍에서 프론트엔드는 컴퓨터 소프트웨어에서 사용자와 상호작용하는 부분을 지칭한다. 그 반대로 백엔드는 사용자의 입력을 받고 처리하는 부분을 말한다. 프론트엔드/백엔드는 거의 대부분 웹 프로그래밍을 지칭한다. 본 개념이 웹에만 존재하는 것은 아니지만 전 세계적으로 프론트엔드 백엔드는 거의 대부분 웹을 통칭한다.

What is Front-end?

프론트엔드는 주로 웹에서 사용자 인터페이스 부분을 명칭한다. 따라서 프론트엔드 개발자들은 백엔드 API에서 가져온 데이터의 출력, 입력을 통한 비즈니스 로직 구성과 사용자와 대화하는 사용자 인터페이스 부분을 작업하는 개발자를 말한다.

프론트엔드 개발은 주로 웹 및 모바일 솔루션의 사용자 인터페이스(UI)와 사용자 경험(UX)을 만드는데 초점을 맞추고 있다. 슬라이더, 드롭다운(drop-down) 메뉴, 폰트, 레이아웃, 컬러 등 모든 요소가 프론트엔드 개발을 이루는 부분이다. 매력적인 프론트엔드를 만드는 데는 HTML, CSS, JavaScript가 중요한 역할을 한다.

Programming Languages

프론트엔드 개발에서 사용하는 언어는 상당히 한정적으로, 대표적으로 HTML, CSS, Javascript가 있다. 간략히 요약하자면, HTML은 웹 페이지의 구조, CSS는 스타일을, JavaScript는 웹의 동적 반응을 담당한다. 이 셋이 현대 웹의 표준으로서 웹을 이루고 있다.

HTML

HTML(Hyper Text Markup Language) 이름 그대로 하이퍼텍스트와 마크업 언어로 구성되어 있다.

하이퍼텍스트는 웹 페이지들 사이의 연결(Hyperlink)를 나타낸다. 반면 마크업 언어는 데이터를 기술하는 언어이다. 한마디로 문서가 화면에 표시되는 형식을 나타내거나 데이터의 논리적인 구조를 명시하기 위한 규칙들을 정의한 언어의 일종이라 할 수 있다. 그리하여 웹 페이지에 문단, 동영상, 제목, 표 등을 정의하고 그 구조와 의미를 정의하는 데 사용된다.

CSS

CSS 종속 스타일 시트(Cascading Style Sheet)의 약자이다.

스타일 시트라는 단어에서 유추할 수 있듯이 CSS는 마크업으로 작성된 것이 실제로 화면에 어떻게 보이는지 그 스타일(표현)을 정해 주는 언어이다. 즉, HTML 콘텐츠를 크기, 색상, 위치 변경 등으로 예쁘게 꾸미는 디자인 역할을 수행한다. CSS의 사용으로 구조와 표현이 함께 움직이는 것이 아닌 서로 분리될 수도 있다. CSS는 개발자가 웹페이지에 다양한 스타일을 적용할 수 있게 해줌으로써 애플리케이션 페이지를 표현하는 프로세스를 보다 단순하게 만들어 줍니다. CSS는 또한 한번에 여러 웹 페이지의 레이아웃을 제어할 수 있어 일일이 디자인을 적용해야 할 수고로움을 덜어둔다.

JavaScript

JavaScript HTML과 CSS를 통해 만들어진 웹 페이지가 사용자와 상호작용하여 동작할 수 있도록 만들어주는 언어이다.

사용자들을 위해 상호작용하는 애플리케이션은 JavaScript를 통해 구현 가능하게 된 것이라 볼 수 있다. JavaScript는 어마어마한 인기를 가진 언어로써, 객체 기반 언어이기에 객체 지향형 프로그래밍과 함수형 프로그래밍 모두를 표현할 수 있다. 게다가 동적이며 소스 코드를 직접 해석하여 바로 실행할 수 있는 인터프리터 언어이기 때문에 타입을 명시할 필요가 없다는 특성을 가지고 있다.

그렇기에 웹 사이트의 기능성을 향상하는데 사용되며, 웹 기반의 소프트웨어 또는 게임들을 실행할 수 있게 해 주는 역할을 한다.
